# 노션 활동 템플릿 가이드

이 문서는 운영자가 노션에서 활동 콘텐츠를 작성할 때 사용하는 템플릿 가이드입니다.

## Story 2.5: 운영자 경험 개선

일관된 형식으로 활동을 작성하여 실수 없이 안정적으로 콘텐츠를 관리할 수 있습니다.

---

## 📋 템플릿 구조

### 필수 필드 (Required Fields)

다음 필드는 **반드시** 입력해야 합니다. 필수 필드가 없으면 활동이 사이트에 표시되지 않습니다.

| 필드명 (한글/영문) | 타입 | 설명 | 예시 |
|-------------------|------|------|------|
| **제목** / **Title** | Title | 활동 제목 | "2026년 1월 정기 모임" |
| **날짜** / **Date** | Date | 활동 날짜 | 2026-01-15 |
| **요약** / **Summary** | Text / Rich Text | 활동 요약 (간단한 설명) | "월간 정기 모임을 진행했습니다." |
| **본문** / **Body** | Text / Rich Text 또는 페이지 블록 | 활동 본문 (상세 내용) | 마크다운 형식으로 작성 가능 |
| **공개 여부** / **Published** | Checkbox | 공개/비공개 설정 | ☑️ 체크 = 공개, ☐ 미체크 = 비공개 |

**참고:**
- **슬러그** / **Slug** 필드는 선택사항입니다. 없으면 자동으로 제목에서 생성됩니다.
- **공개 여부**가 없거나 체크되어 있으면 공개로 처리됩니다.

### 선택적 필드 (Optional Fields)

다음 필드는 선택사항입니다. 없어도 활동이 정상적으로 표시됩니다.

| 필드명 (한글/영문) | 타입 | 설명 | 예시 |
|-------------------|------|------|------|
| **카테고리** / **Category** | Select | 활동 카테고리 | "모임", "서비스", "행사" 등 |
| **이미지** / **Image** | Files | 활동 대표 이미지 | 이미지 파일 업로드 |

## 📝 템플릿 사용 방법

### 1. 새 활동 작성

1. 노션 데이터베이스에서 새 페이지를 생성합니다.
2. 다음 필드를 순서대로 작성합니다:

   ```
   제목: [활동 제목 입력]
   날짜: [활동 날짜 선택]
   요약: [활동 요약 입력]
   본문: [활동 본문 입력 또는 페이지 블록으로 작성]
   공개 여부: ☑️ (공개하려면 체크)
   카테고리: [선택사항] [카테고리 선택]
   이미지: [선택사항] [이미지 업로드]
   ```

### 2. 본문 작성 방법

본문은 두 가지 방법으로 작성할 수 있습니다:

#### 방법 A: 필드에 직접 입력
- **본문** 필드에 텍스트를 직접 입력합니다.
- 마크다운 형식을 사용할 수 있습니다.

#### 방법 B: 페이지 블록으로 작성 (권장)
- 페이지 본문에 직접 콘텐츠를 작성합니다.
- 마크다운 형식이 자동으로 변환됩니다.
- 더 풍부한 콘텐츠 작성이 가능합니다.

**마크다운 지원 형식:**
- 헤더: `# 제목`, `## 소제목`, `### 소소제목`
- 굵게: `**텍스트**`
- 기울임: `*텍스트*`
- 링크: `[텍스트](URL)`
- 리스트: `- 항목`
- 코드 블록: ` ```코드``` `

### 3. 공개/비공개 설정

- **공개**: 체크박스를 체크하거나 Status를 "공개"로 설정
- **비공개**: 체크박스를 체크하지 않거나 Status를 "비공개"로 설정
- **기본값**: 필드가 없으면 자동으로 공개로 처리

## ✅ 필드 작성 체크리스트

활동을 작성할 때 다음을 확인하세요:

- [ ] **제목**이 입력되어 있나요?
- [ ] **날짜**가 올바르게 설정되어 있나요?
- [ ] **요약**이 입력되어 있나요? (선택사항이지만 권장)
- [ ] **본문**이 입력되어 있나요? (필드 또는 페이지 블록)
- [ ] **공개 여부**가 올바르게 설정되어 있나요?
- [ ] **슬러그**가 고유한가요? (자동 생성되므로 보통 확인 불필요)

## ⚠️ 주의사항

### 필수 필드 누락 시

필수 필드(제목, 날짜)가 없으면:
- 활동이 사이트에 표시되지 않습니다.
- GitHub Actions 로그에 경고 메시지가 표시됩니다.
- 다른 활동은 정상적으로 표시됩니다.

### 잘못된 형식 데이터 입력 시

시스템은 다음을 자동으로 처리합니다:
- 날짜 형식이 잘못된 경우: 경고 후 해당 활동 제외
- 제목이 비어있는 경우: 해당 활동 제외
- 요약이나 본문이 없는 경우: 제목을 기본값으로 사용
- 공개 여부가 불명확한 경우: 기본값(공개)으로 처리

**중요**: 잘못된 형식의 데이터를 입력해도 사이트가 깨지지 않습니다. 해당 활동만 제외되고 나머지는 정상적으로 표시됩니다.

## 📖 필드명 매핑

시스템은 다음 필드명을 자동으로 인식합니다 (대소문자, 공백 무시):

### 필수 필드
- **제목**: `제목`, `Title`, `title`, `이름`, `Name`, `name`
- **날짜**: `날짜`, `Date`, `date`, `일자`, `날짜/시간`
- **요약**: `요약`, `Summary`, `summary`, `설명`, `Description`, `description`
- **본문**: `본문`, `Body`, `body`, `내용`, `Content`, `content`
- **공개 여부**: `공개여부`, `공개 여부`, `Published`, `published`, `Public`, `public`

### 선택적 필드
- **슬러그**: `슬러그`, `Slug`, `slug`, `URL`, `url`
- **카테고리**: `카테고리`, `Category`, `category`, `분류`
- **이미지**: `이미지`, `Image`, `image`, `파일과 미디어`, `Files`, `files`, `파일`, `File`, `file`, `미디어`, `Media`, `media`

## 🔄 동기화 프로세스

1. 노션에서 활동을 작성하거나 수정합니다.
2. GitHub Actions가 5분마다 자동으로 동기화합니다.
3. 변경사항이 사이트에 자동으로 반영됩니다.
4. 수동으로 동기화하려면 GitHub Actions에서 "Run workflow"를 클릭하세요.

## ❓ 자주 묻는 질문 (FAQ)

### Q: 필수 필드를 빠뜨리면 어떻게 되나요?
A: 해당 활동만 사이트에 표시되지 않습니다. 다른 활동은 정상적으로 표시됩니다.

### Q: 날짜 형식은 어떻게 해야 하나요?
A: 노션의 Date 필드를 사용하면 자동으로 올바른 형식으로 처리됩니다.

### Q: 본문에 이미지를 넣을 수 있나요?
A: 네, 페이지 블록으로 작성하면 이미지를 포함할 수 있습니다.

### Q: 공개 여부를 나중에 변경할 수 있나요?
A: 네, 언제든지 노션에서 수정하면 5분 이내에 사이트에 반영됩니다.

### Q: 슬러그는 어떻게 생성되나요?
A: 슬러그 필드가 없으면 제목에서 자동으로 생성됩니다. 예: "2026년 1월 정기 모임" → "2026년-1월-정기-모임"
